ARQUITECTURA DE RENDERIZADO VECTORIAL PARA INFOGRAFÍAS EDUCATIVAS MASIVAS
ESTADO DEL PROYECTO: Sistema definido mediante plantillas HTML/CSS y componentes SVG low-poly inyectados vía JSON con capacidad de renderizado local instantáneo.

La consulta inicial surgió de la necesidad del usuario por integrar el módulo "Máquina del Tiempo" descrito en su documento arquitectónico, buscando un método viable para generar automáticamente cerca de 400 infografías diarias sin saturar el almacenamiento de un servidor Mac Mini local ni depender de exportaciones estáticas de plataformas de IA externas como NotebookLM. La intención implícita era encontrar una infraestructura ligera y escalable capaz de albergar un vasto repositorio histórico accesible de forma cronológica, manteniendo un alto impacto visual educativo. El proceso evolucionó rápidamente tras descartar la idea de almacenar imágenes pesadas, pivoteando hacia una arquitectura de renderizado en vivo basado en plantillas. Esta transición inició con pruebas gráficas rudimentarias de isometría CSS (sistema circulatorio, 400x800px) que no cumplieron las expectativas visuales del usuario, forzando una reevaluación hacia vectores mucho más ambiciosos. El giro técnico definitivo ocurrió al migrar hacia gráficos SVG codificados manualmente con estética "low-poly" hiperrealista, ambient occlusion y efectos de luz integrados. Adicionalmente, se acordó abandonar las imágenes rasterizadas y estructurar un flujo donde la Inteligencia Artificial diseña vectores SVG embebidos que flotan junto a texto envolvente (CSS shape-outside) sobre una dimensión estricta de 500x700px optimizada para lectura en dispositivos y exportación futura. La solución final establece un paradigma de "fábrica": se almacenan únicamente registros puros de texto y rutas alusivas dentro de una base de datos local (netmarlyn_registros.json), que luego son inyectados dinámicamente en plantillas maestras HTML/CSS. Esto permite renderizaciones masivas ultra-ligeras con resoluciones infinitas, y la capacidad comprobada de exportar vía cliente (html2canvas) dichas estructuras vectoriales a formatos PDF o PNG imprimibles para el uso de los alumnos. El marco de exportación y proyección inmersiva queda documentado, aunque la implementación en código de sus librerías correspondientes permanece pendiente de definición técnica para las siguientes fases de desarrollo del entorno PWA.

---

Para ejecutar esta arquitectura y escalar la producción, se requiere implementar el motor de plantillas y definir esquemas de datos. [1] (40%) Desarrollar un script en JavaScript que conecte netmarlyn_registros.json con una plantilla HTML vacía, inyectando variables estructuradas (title, svg_code, content) para validar el renderizado dinámico en el DOM. [2] (35%) Diseñar la interfaz y el modelo de datos JSON estandarizado que las IAs generadoras deberán seguir al redactar el texto y codificar las coordenadas SVG low-poly requeridas para cada nueva lección educacional. [3] (25%) Integrar la funcionalidad de exportación del lado del cliente utilizando bibliotecas como html2canvas o jspdf acopladas a botones de "Descargar", realizando pruebas de calidad sobre las exportaciones de canvas SVG para garantizar fidelidad cromática.
